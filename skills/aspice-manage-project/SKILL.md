# 项目管理

## 执行步骤

1. 完整读取 [ASPICE 文档契约](references/document-contract.md)、[过程参考](references/process.md) 和 [文档模板](assets/template.md)
   - 完成标准：已明确计划基线、进度状态和项目完成结论的边界
2. 识别项目目标和可用计划输入
   - 优先读取 PRD 或目标、交付范围、仓库现状、实际人员、关键依赖和已有问题；合同、历史估算和组织流程存在时再纳入
   - 缺少人员姓名或精确可用时间时记录计划假设和确认行动，只有确实无法分配近期工作或作出授权决定时才形成阻塞
   - 完成标准：目标、约束、可用能力和影响近期计划的未知项已识别
3. 定义工作范围和生命周期
   - 记录目标、动机、边界、交付物、阶段、里程碑和各里程碑发布范围
   - 完成标准：范围内外明确，使用与项目规模相称的阶段和里程碑
4. 评估可行性并定义工作包
   - 基于技术约束、工作量、资源、技能和依赖评估可行性
   - 将近期工作拆为可监控的工作包，使用区间、相对大小或历史数据估算
   - 完成标准：每项交付物有工作包承接，估算不伪造精度，真实资源或技能缺口可见
5. 定义项目接口、承诺和进度计划
   - 只记录实际参与方、输入输出、承诺、依赖和里程碑日期；一人项目由项目负责人直接决策和调整
   - 完成标准：关键依赖有实际负责人，存在外部接口时有沟通和升级方式
6. 监控一致性并评审项目进展
   - 对照范围同步调整估算、资源、技能、工作包、依赖和进度
   - 对偏差记录纠正措施，连接问题或变更请求
   - 完成标准：近期和进行中的工作包有最新状态，实际偏差有处置和责任人
7. 创建或更新 `project-management.md`，执行交付检查并报告路径、结论、偏差和下一里程碑
   - 完成标准：计划足以支持下一里程碑决策，没有企业组织假设和虚假精确估算
