# MAN.3 项目管理

来源：Automotive SPICE PAM 4.0 中文版，第 100～102 页

## 目的

在项目需求和约束下识别并控制产品开发活动，建立所需资源

## 必须覆盖的成果

- 定义项目目标、动机、边界和生命周期
- 依据时间、估算、资源和技术约束评估可行性
- 定义并监控工作包、依赖、工作量和资源估算
- 定义并监控所需技能、知识和经验
- 识别项目接口、约定承诺和升级路径
- 为工作包分配资源并维护进度计划
- 保持范围、估算、资源、技能、依赖和计划一致
- 定期评审和报告进展，对偏差采取纠正措施

## 证据重点

- 工作范围和生命周期
- 可行性分析、工作包、估算、资源和进度
- 参与方、接口、承诺及升级路径
- 项目状态、偏差、问题、变更和纠正措施
