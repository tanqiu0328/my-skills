# 变更请求管理

## 执行步骤

1. 完整读取 [ASPICE 文档契约](../../references/aspice-document-contract.md)、[过程参考](references/process.md) 和 [文档模板](assets/template.md)
   - 完成标准：已明确输出位置、证据规则和 SUP.10 全部过程成果
2. 检查变更来源、关联问题、需求、设计、代码、测试、配置基线、发布计划和历史变更
   - 完成标准：变更动机、当前状态和候选影响范围已确认
3. 唯一标识并记录变更请求
   - 记录发起者、原因、范围、目标版本、状态和预期收益
   - 完成标准：变更边界明确，相关方对要改变什么有共同理解
4. 分析和评估影响
   - 识别受影响工作产品、其他变更依赖、资源、进度、风险和验证需求
   - 完成标准：所有受影响工作产品均已纳入分析或说明不适用理由
5. 在实施前记录优先级和批准结论
   - 记录批准人、日期、条件和目标发布范围
   - 完成标准：实施授权可追溯，未批准变更没有进入实现状态
6. 跟踪实施并确认结果
   - 建立变更与问题、需求、设计、代码、测试和配置项的双向追溯
   - 连接实现、评审和验证证据，由相关方确认后关闭
   - 完成标准：每项已实施变更都有验证结果，未完成项有责任人和下一行动
7. 创建或更新 `change-log.md`，执行交付检查并报告路径、影响、决定和状态
   - 完成标准：契约中的每项交付检查均通过
