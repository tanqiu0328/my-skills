# SUP.10 变更请求管理

依据：[Automotive SPICE PAM 4.0 中文版](../../../references/AutomotiveSPICE_PAM_40_Chinese.pdf) 第 95～96 页

## 目的

确保变更请求被记录、分析、跟踪、批准和实施

## 必须覆盖的成果

- 唯一标识并记录变更发起者、原因、范围和状态
- 分析受影响工作产品、其他变更依赖、资源、进度、风险与收益
- 在实施前确定优先级并记录批准
- 保持变更与受影响工作产品及来源问题的双向追溯
- 由相关方确认变更实施
- 跟踪至关闭并向受影响方沟通状态

## 证据重点

- 变更请求和分析准则
- 影响分析、优先级及批准记录
- 需求、设计、代码、测试、配置和问题的追溯
- 实施、验证、确认与关闭状态
