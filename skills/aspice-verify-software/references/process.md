# SWE.6 软件验证

来源：Automotive SPICE PAM 4.0 中文版，第 62～63 页

## 目的

验证集成软件与软件需求一致

## 必须覆盖的成果

- 基于功能和非功能软件需求定义验证措施
- 按发布范围、优先级、变更影响和回归准则选择验证措施
- 对集成软件执行验证并记录结果
- 保持软件需求、验证措施和结果之间的一致性及双向追溯
- 汇总结果并与受影响方沟通

## 每项验证措施

- 验证技术、顺序和对象
- 通过和失败准则
- 准入和准出准则
- 基础设施、环境和测试数据
- 执行软件基线与原始结果

偏离预期的结果连接 SUP.9 问题记录

## 小团队适配

- 每项发布范围内软件需求仍需验证措施，但措施可以复用同一端到端场景或检查证据
- 验证计划可以先形成基线；只有选定措施执行完成并处理偏差后才能给出发布结论
- 回归集按变更影响和风险选择，不要求每次发布重复全部历史测试
- 尚未执行的措施标记为已规划，失败和环境问题只阻塞受影响需求或发布范围
- 覆盖率、通过率和发布状态分别报告，计划覆盖不能替代执行结果
