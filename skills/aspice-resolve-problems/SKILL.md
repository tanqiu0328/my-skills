# 问题解决管理

## 执行步骤

1. 完整读取 [ASPICE 文档契约](references/document-contract.md)、[过程参考](references/process.md) 和 [文档模板](assets/template.md)
   - 完成标准：已明确问题记录、分析、解决和关闭的证据边界
2. 识别本次问题范围和已有证据
   - 先读取问题报告、复现环境和直接日志，再按影响检查相关代码、测试、变更和同类问题
   - 完成标准：问题来源、当前状态和继续诊断所需的证据已确认
3. 唯一标识并记录问题
   - 记录症状、复现步骤、环境、发现来源、严重度、紧急度和负责人
   - 完成标准：他人能够使用记录的信息复现或继续诊断问题
4. 分析原因和影响
   - 始终分析直接原因和影响；高严重度、重复或系统性问题再展开根因和共性原因
   - 完成标准：结论有证据，假设与已证实原因分开，分析深度与风险相称
5. 启动并跟踪解决
   - 按需要记录紧急措施、授权和通知，并为需要修改受控工作产品的问题建立长期行动或变更请求
   - 完成标准：实际需要的行动有负责人和状态，临时措施不会替代长期修复
6. 验证并关闭问题
   - 连接修复提交、评审、回归结果和受影响方接受证据
   - 完成标准：问题只有在解决方案得到验证且相关方接受后才标记为关闭
7. 汇总状态与趋势，创建或更新 `problem-log.md`
   - 首屏保留总体状态；重复问题使用不超过五列的索引表，复杂问题再展开详情
   - 完成标准：范围内问题可追踪到关闭或下一行动；样本不足时明确“暂无趋势结论”
