# SWE.2 软件架构设计

依据：[Automotive SPICE PAM 4.0 中文版](../../../references/AutomotiveSPICE_PAM_40_Chinese.pdf) 第 52～53 页

## 目的

建立与软件需求一致、经过分析且包含静态和动态方面的软件架构

## 必须覆盖的成果

- 定义组件、外部接口、组件接口和依赖等静态方面
- 定义模式、行为、交互、时序和并发等动态方面
- 分析技术设计并记录关键设计依据
- 保持软件架构与软件需求的一致性和双向追溯
- 与受影响方约定并沟通软件架构

## 分析重点

- 功能分配与接口边界
- ROM、RAM、CPU、存储和时序
- 中断、抢占、任务、线程和模式切换
- 复用、平台适用性、自研或外购
- 原型、仿真或定性分析形成的证据
