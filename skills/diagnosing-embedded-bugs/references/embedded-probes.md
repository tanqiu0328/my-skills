# 嵌入式诊断探针

只读取与当前假设相关的章节，每个探针都必须产生能够证实或推翻假设的观测

## MCU 异常与复位

- 保存 fault 类型、堆栈指针、堆叠寄存器、`CFSR`、`HFSR`、`MMFAR`、`BFAR` 和复位原因
- 使用 ELF、map 文件和精确固件提交把 PC/LR 映射到源码
- 区分本次异常与上次复位遗留标志，确认复位标志清除时点
- 对栈破坏使用 guard、watermark、MPU 或 watchpoint，不依赖崩溃位置猜测写坏位置

## 内存与数值

- 对堆使用分配失败注入、所有权追踪和高水位；安全关键路径优先检查是否允许运行时动态分配
- 对栈记录每个任务高水位和最深调用链，不用单次空闲量代表最坏情况
- 对边界计算检查整数提升、截断、符号转换、溢出和单位换算
- 对疑似越界写入使用硬件 watchpoint、MPU、ASan 主机构建或带 canary 的测试构建

## ISR、RTOS 与竞态

- 记录执行上下文、优先级、临界区、锁顺序和共享对象所有权
- `volatile` 只保证可观察访问，不证明原子性、互斥或内存顺序
- 检查 ISR 可调用 API、FromISR 变体、优先级上限和上下文切换请求
- 用事件序列号、trace 或时间戳重建先后关系，避免用大量日志改变竞态窗口

## DMA、Cache 与外设

- 核对 DMA 可访问内存区、地址和长度对齐、传输方向以及 Cache clean/invalidate 的时点
- 对环形缓冲区明确生产者/消费者数量、索引原子性、回绕规则和发布顺序
- 从 HAL 调用追到寄存器、时钟、引脚复用、中断状态和错误标志
- 使用逻辑分析仪或示波器时同时保存固件时间戳，建立软硬件事件对应关系

## 实时性与性能

- 先定义截止时间、测量点、时钟源和统计口径
- 同时观察平均值、最坏值、抖动、队列深度、CPU 负载和中断占用
- 优先使用 trace、周期计数器和采样分析；日志只用于区分少量假设
- 对回归执行版本、配置和负载二分，保持测量环境一致
