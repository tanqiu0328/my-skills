# SWE.1 软件需求分析

来源：Automotive SPICE PAM 4.0 中文版，第 50～51 页

## 目的

建立一组经过结构化和分析、与系统需求及系统架构一致的软件需求

## 必须覆盖的成果

- 定义功能性和非功能性软件需求
- 结构化需求并确定优先级
- 分析正确性、技术可行性和相互依赖
- 分析需求对运行环境的影响
- 保持软件需求与系统需求、系统架构的一致性和双向追溯
- 与受影响方约定并沟通软件需求和环境影响

## 证据重点

- 需求及其属性
- 正确性、可行性和运行环境影响分析
- 上游需求、架构与软件需求之间的一致性证据
- 受影响方可获得约定内容的证据

追溯链接只证明关系存在，还要检查上下游含义是否一致
