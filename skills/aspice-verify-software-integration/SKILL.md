# 软件组件验证与集成验证

## 执行步骤

1. 完整读取 [ASPICE 文档契约](references/document-contract.md)、[过程参考](references/process.md) 和 [文档模板](assets/template.md)
   - 完成标准：已明确集成计划、执行结果和完成结论的边界
2. 确定集成分支和范围
   - 区分制定集成计划、执行选定步骤或汇总结果，并读取相应的架构、详细设计、组件、接口和已有验证证据
   - 完成标准：待集成要素、软件版本和本次交付状态已明确
3. 定义集成顺序和前提条件
   - 说明逐步集成、持续集成或一次性集成的选择及依赖关系
   - 完成标准：每个范围内软件要素都有适合项目规模的集成时点、依赖和必要准入条件
4. 定义组件验证与集成验证措施
   - 覆盖架构中关键的组件行为、接口解释、数据流、动态交互，以及适用的时序和资源目标
   - 写明技术、环境、通过/失败、准入/准出和回归准则
   - 完成标准：架构和详细设计中的关键静态及动态关系均有验证措施
5. 选择并执行验证措施
   - 用户只要求计划时将步骤和措施标记为“已规划”，不把尚未执行写成阻塞
   - 用户授权执行时保存每个执行步骤和组件验证的结果、数据及异常链接
   - 完成标准：执行分支中的每项已选措施都有结果或局部受阻原因，失败项连接到问题记录
6. 建立架构、详细设计、验证措施与结果的双向追溯，汇总并沟通结论
   - 完成标准：计划分支可从设计定位措施，执行分支还能从结果反查设计依据
7. 创建或更新 `integration-verification.md`，执行交付检查并报告路径、集成状态和阻塞项
   - 完成标准：文档明确是集成计划、执行中记录或完成报告，结论不超出证据
