# SWE.5 软件组件验证与集成验证

来源：Automotive SPICE PAM 4.0 中文版，第 58～61 页

## 目的

验证组件与架构一致，集成软件要素，并验证集成结果与架构及详细设计一致

## 必须覆盖的成果

- 按架构、详细设计、接口和交互定义集成验证措施
- 为软件组件行为和接口定义组件验证措施
- 按顺序和前提条件集成软件要素
- 按发布范围和回归准则选择验证措施
- 执行组件验证和集成验证并记录结果
- 保持设计、验证措施和结果之间的一致性及双向追溯
- 汇总结果并与受影响方沟通

## 验证重点

- 接口数据的正确解释与传递
- 组件动态交互和时序依赖
- 软件组件行为
- 资源消耗目标
- 每个集成步骤的准入、准出和环境

偏离预期的结果连接 SUP.9 问题记录

## 小团队适配

- 集成策略按项目规模选择，少量强耦合组件可以一次集成，仍需记录前提条件和接口验证
- 只验证架构实际定义的关键接口、动态关系、时序和资源目标，不机械展开全部类别
- 单元验证未全部结束时，可对满足局部准入条件的组件逐步集成，不扩大为整个集成计划受阻
- 集成计划可以先形成可用基线，实际执行后再给出组件和集成完成结论
- 尚未执行的步骤标记为已规划，失败或环境不可用只阻塞对应步骤
