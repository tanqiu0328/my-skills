# SWE.3 软件详细设计与单元构建

来源：Automotive SPICE PAM 4.0 中文版，第 54～56 页

## 目的

建立与软件架构一致的静态和动态详细设计，并构建与详细设计一致的软件单元

## 必须覆盖的成果

- 定义软件单元的静态结构、关系、接口、有效值域和单位
- 定义软件单元间交互和组件动态行为
- 按编码原则开发与设计一致的软件单元
- 保持需求、架构、详细设计和代码之间的一致性及双向追溯
- 与受影响方沟通约定的详细设计和已开发单元

## 证据重点

- 软件详细设计
- 软件单元或模型实现
- 需求、架构、设计和代码之间的一致性证据
- 受影响方可获得设计和实现的证据

软件单元由应用语义决定，不等同于单个源文件或函数

## 小团队适配

- 详细程度按单元复杂度和风险决定，状态机、并发、硬件访问、公共 API 和错误恢复优先展开
- 简单封装、配置表和胶水代码可通过职责、接口约束和代码链接形成证据，不逐行复述实现
- 尚未编码不阻塞详细设计基线，使用实现状态和行动项区分设计完成度与构建进度
- 资源所有权、有效范围、单位和错误契约只在适用时填写，不制造空洞章节
- 同一工程师可以完成设计、实现和自检，但设计依据、代码链接和验证入口仍分别可追溯
