# 配置管理

## 执行步骤

1. 完整读取 [ASPICE 文档契约](references/document-contract.md)、[过程参考](references/process.md) 和 [文档模板](assets/template.md)
   - 完成标准：已明确开发基线、发布基线和 SUP.8 完成结论的边界
2. 识别当前配置管理范围
   - 优先检查代码仓库、工具链、构建命令、依赖和当前交付物；CI、制品库、权限和备份机制存在或影响发布时再检查
   - 完成标准：影响当前构建或交付重现的输入、位置和工具已定位，未使用的企业设施未被列为缺口
3. 识别配置项并定义属性
   - 按类别记录需要受控的代码、配置、文档、工具、依赖和制品，同一负责人可管理多个类别
   - 完成标准：会改变当前目标输出的配置项均已纳入，其他内容按明确准则排除
4. 定义并检查修改控制
   - 说明项目实际使用的分支、提交、评审、标签、权限和状态规则
   - 完成标准：每类配置项有可追溯的修改入口和历史记录；未使用的控制机制标记为不适用
5. 建立或核对基线
   - 记录基线范围、版本、提交、构建参数、依赖和制品校验信息
   - 完成标准：开发或发布基线类型明确，并能使用记录的受控输入和命令重现对应输出
6. 检查配置状态以及备份和恢复可用性
   - 用户授权恢复演练时执行并保留结果；未授权时记录现有机制、恢复步骤和验证阶段门槛
   - 完成标准：配置状态可汇报；SUP.8 标记完成前具有备份恢复验证证据，开发基线可先独立建立
7. 创建或更新 `configuration-management.md`，执行交付检查并报告路径、基线状态和缺口
   - 完成标准：文档用最少规则说明配置项、修改控制、基线重现和备份恢复状态
