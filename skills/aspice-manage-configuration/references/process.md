# SUP.8 配置管理

来源：Automotive SPICE PAM 4.0 中文版，第 90～92 页

## 目的

建立和维护配置项及基线的完整性，并使其对受影响方可用

## 必须覆盖的成果

- 定义选择准则并识别配置项
- 定义配置项的状态、位置、权限等属性
- 建立配置管理及并行修改控制机制
- 控制配置项修改
- 建立内部和交付基线
- 记录、汇总和沟通配置状态
- 确保配置项及基线完整一致
- 验证备份和恢复机制可用

## 证据重点

- 配置项选择准则和清单
- 分支、合并、评审、访问和状态控制
- 基线范围、版本和构建输入
- 修改历史和配置状态
- 完整性检查及备份恢复验证

## 小团队适配

- 最小配置范围通常包括仓库提交、工具链版本、构建命令、关键配置、依赖版本和交付制品
- 单仓库、单负责人和简单分支策略可以满足当前项目，不要求引入未使用的 CI、制品平台或权限矩阵
- 开发基线和发布基线分开判断，前者可以早于正式制品和恢复演练建立
- Git 远端或其他副本可以作为备份机制的一部分，SUP.8 完成仍需一次可追溯的恢复验证
- 配置项按类别管理，避免为每个普通源文件建立独立记录
