# 软件单元验证

## 执行步骤

1. 完整读取 [ASPICE 文档契约](references/document-contract.md)、[过程参考](references/process.md) 和 [文档模板](assets/template.md)
   - 完成标准：已明确验证计划、执行结果和完成结论的边界
2. 确定验证分支和范围
   - 区分仅制定计划、选择本次执行集或汇总执行结果，并读取相应的详细设计、软件单元和已有证据
   - 完成标准：本次范围、软件版本和期望交付状态已明确
3. 定义单元验证措施
   - 为每个范围内单元按风险选择静态分析、代码评审、单元测试或适用组合
   - 写明通过/失败、准入/准出、环境、数据和覆盖准则
   - 完成标准：每个范围内单元至少有一项足以验证其详细设计的措施，不强制叠加所有技术
4. 根据发布范围和变更影响选择本次执行集
   - 记录选择依据与回归准则
   - 完成标准：发布或变更影响范围内的单元均已选择，排除项有明确理由
5. 执行或收集验证结果
   - 用户只要求计划时将措施标记为“已规划”，不把尚未执行写成阻塞
   - 用户授权执行时运行适用工具，保存命令、环境、版本、通过/失败状态及原始结果链接
   - 完成标准：执行分支中的每项已选措施都有结果或局部受阻原因，异常连接到问题记录
6. 建立单元、验证措施与结果的双向追溯，汇总并沟通结论
   - 完成标准：计划分支可从单元找到措施，执行分支还能从结果反查单元和措施
7. 创建或更新 `unit-verification.md`，执行交付检查并报告路径、覆盖结论和阻塞项
   - 完成标准：文档明确是验证计划、执行中记录或完成报告，结论不超出证据
