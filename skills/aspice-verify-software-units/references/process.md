# SWE.4 软件单元验证

来源：Automotive SPICE PAM 4.0 中文版，第 56～58 页

## 目的

验证软件单元与软件详细设计一致

## 必须覆盖的成果

- 为详细设计中的每个软件单元定义验证措施
- 按发布范围和回归准则选择验证措施
- 执行静态分析、代码评审、单元测试或适用组合并记录结果
- 保持单元、验证措施和结果之间的一致性及双向追溯
- 汇总结果并与受影响方沟通

## 每项验证措施

- 验证技术与对象
- 通过和失败准则
- 准入和准出准则
- 所需基础设施与数据
- 选择依据与回归准则
- 执行版本、结果和原始证据

偏离预期的结果连接 SUP.9 问题记录

## 小团队适配

- 每个范围内单元需要适合其风险的验证措施，不要求同时使用静态分析、评审和单元测试
- 纯配置或简单封装可用检查或评审，状态机、算法、边界计算和错误处理优先使用单元测试
- 覆盖率目标来自项目约定或风险分析；没有目标时报告实际数据或“不适用”，不编造阈值
- 验证计划可以先形成可用基线，只有实际执行并处理失败后才能给出完成结论
- 尚未进入本次执行集的措施标记为已规划或未选择，不扩大为整份文档受阻
