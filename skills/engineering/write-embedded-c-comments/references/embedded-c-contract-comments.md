# 嵌入式 C 契约注释

仅在目标代码涉及对应场景时应用本文件。先检查实现、调用链和项目配置，
再写契约；没有证据时报告未知项，不使用经验值补全。

## 回调与 ISR

- 注明回调运行在 ISR、工作线程、调用者线程还是主循环上下文。
- 注明回调是否必须保持简短、是否允许阻塞，以及允许调用的 API 类别。
- 注明回调数据在返回后是否仍然有效，以及注册与注销的生命周期要求。
- 对 ISR 注明项目要求的中断优先级或优先级来源；不要编造具体数值。
- 仅在实现确实如此时，说明 ISR 只设置标志、释放信号量或投递队列。

## 线程安全、可重入性与临界区

- 明确函数是否线程安全、是否可重入、可从哪些上下文调用。
- 非线程安全或不可重入时，说明调用者必须采用的串行化方式。
- 对共享状态说明保护机制；对临界区说明必须保护的状态和使用原因。
- 不要仅凭使用互斥锁或 `volatile` 宣称线程安全，必须检查完整访问路径。

## 异步与非阻塞接口

- 说明调用是否立即返回、请求是否入队，以及完成或错误如何通知。
- 说明队列满、忙碌、超时和未初始化时的实际返回结果。
- 说明输入缓冲区是复制还是借用，以及调用返回后的有效期要求。
- 沿硬件调用链验证阻塞性；函数名带有 `Async` 不能作为证据。

## 缓冲区、所有权与资源生命周期

- 注明缓冲区方向、长度单位、最大值、可否为 `NULL` 和对齐要求。
- 注明资源由谁创建、持有、释放，所有权是否转移，以及失败路径责任。
- 注明初始化、打开、关闭、销毁和回调注销的必要顺序。
- 对调用者提供的存储，说明函数是否保存指针及其最短有效期。

## 硬件与时序

- 注明已验证的阻塞行为、超时条件和硬件副作用。
- 注明调用前置条件，例如时钟、GPIO、DMA、缓存或外设状态要求。
- 注明寄存器访问顺序、必要延时和读改写约束，但不复述每条寄存器语句。
- 涉及 DMA 与 Cache 时，说明缓冲区一致性和 Cache 维护责任。

## Doxygen 契约示例

```c
/**
 * @brief  将发送请求加入驱动工作队列
 * @param  self  驱动实例，不能为 NULL
 * @param  data  待发送缓冲区；函数返回前完成数据复制
 * @param  len   待发送字节数，最大为 TX_BUF_SIZE
 * @return 成功返回 ERR_OK；队列已满时返回 ERR_QUEUE_FULL
 * @note   本函数非阻塞且线程安全；发送结果在工作线程回调中通知
 */
int Driver_SendAsync(Driver_t *self,
                     const uint8_t *data,
                     uint16_t len);
```

示例只展示格式，不能作为目标代码契约的证据。
